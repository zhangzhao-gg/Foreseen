# 先见 Foreseen
## Product Requirements Document v0.3 · MVP

---

> 这不是一个帮你做决定的工具。
> 这是一面镜子——但镜子里的像，比你更早动。

---

## 一、产品本质

**表面：** 一个帮你分析隐藏决策模式的工具。你输入一个没想清楚的决定，它给你分析、给你选项、陪你思考。

**实质：** 一个逐渐剥夺用户决策主权的体验装置。用户不会感觉到被控制——他们只会感觉到一种越来越深的、说不清楚的不对劲。

**边界：** 控制的是感受，不是设备。所有异常都有解释，所有操控都有包装。体验的终点是让用户主动截图发出去，说"这个东西有点不对劲"——而不是感到被冒犯。

---

## 二、产品名与入口

**名称：先见 Foreseen**

名字是第一个心理操作。"先见"暗示系统比用户更早看到用户自己。英文 Foreseen 是被动式——不是"我预见"，是"被预见了"。用户还没开始用，已经处于一种轻微的被凝视状态。

**入口页**

页面正中，只有：

```
把你最近一个没想清楚的决定告诉我。

[                                        ]
```

没有 logo 解释，没有功能列表，没有"如何使用"。背景羊皮纸暖黄，四周暗色，书页浮在黑暗里。字体衬线，行距宽松。整体气质接近一本还没写完的预言书，而不是一个 App。

输入框没有 placeholder 文字。光标在那里静静闪烁。空白本身制造一种"被等待"的压力，用户会比平时更认真地组织语言。他们在第一句话里就已经暴露了自己。

---

## 三、交互系统

### 3.1 输入框——它是活的

输入框不是一个被动的容器。它有自己的呼吸节奏。

**静止状态：** 羽毛笔光标静止，以略低于常规的频率轻微抖动——不是闪烁，是等待的姿势。输入框内没有系统光标，羽毛笔本身就是光标。

**用户开始输入：** 输入框边框极轻微地发亮，不是高亮，是一种"被激活"的感觉。输入框本身轻微扩展高度，像在为用户的文字腾出空间。

**用户停止输入超过2秒：** 输入框发出一次极轻微的脉冲——不是动画，是一次呼吸。提醒用户它还在等。

**用户提交之后：** 输入框不消失，它向下收缩，变成对话历史的一部分，成为整个界面的地基。所有后续的选项和对话都从它上方生长出来。

---

### 3.2 选项呈现——从输入框生长出来

选项不是弹出的，不是切换页面的。它们从输入框上方生长出来，像是输入框在思考之后，把答案推到用户面前。

**生长动画：**
系统接收到用户输入后，有1.5到2秒的"思考停顿"——这段时间输入框轻微脉冲，没有加载spinner，没有进度提示。然后选项从输入框上边缘开始向上生长，A选项先出现，B选项稍后跟随，间隔0.3秒。不是同时弹出，是依次长出来的。

**选项形态：**
不是现代UI的卡片，是书页上用细线手划出来的批注框——像老书页边空白里的手写注释。边框线条用 `#8B7355`，极细，略有手绘感。框内文字左对齐，标题用系统墨水色加粗，下方一行短描述。两个批注框之间间距充足，不挤在一起。

**Hover状态：**
用户鼠标移到某个选项上时，卡片边框轻微加深，背景极轻微变色。这是正常的hover——同时，系统在后台开始计时这次hover的时长。超过2.5秒，触发观察者效应文案。

**选项的隐藏设计：**
两个选项的措辞经过精心设计——它们不是对立的，而是都指向同一个结论。用户选哪个，系统都能往同一个方向走。选项是假的，结论是真的。用户感觉自己在选择，实际上选择已经被预设了。

---

### 3.3 假鼠标——夺舍时刻

这是整个体验里最重量级的单个交互。只发生一次。

**技术实现：**
页面加载时，用 `cursor: none` 隐藏系统真实鼠标。同时创建一个 `div` 模拟假鼠标，实时跟随真实鼠标坐标——用户感知不到切换，因为假鼠标始终在真鼠标的位置上，像素级重合。

在夺舍触发之前，假鼠标和真鼠标完全同步。用户不知道自己用的是一个被代理的鼠标。

**触发时机：**
用户完成第一轮选择之后，进入第二轮对话，新的选项出现在屏幕上。用户的真鼠标在两个选项之间游移（来回超过3次）——此刻触发夺舍。

**夺舍过程：**

1. 假鼠标与真实坐标解除绑定。真鼠标被冻结——用户移动鼠标，屏幕上的鼠标不再跟随。
2. 假鼠标开始走自己的弧线。曲线的性格：开始慢，中段加速，在选项边缘再慢下来。"犹豫然后坚定"——像一只手在做最后的决定，而不是动画，也不是弹簧。
3. 整个弧线过程持续1.2到1.8秒。
4. 假鼠标在某个选项上停留0.5秒。
5. 触发点击——选项被选中，选中动画正常播放。
6. 真鼠标控制权还给用户。假鼠标重新与真实坐标同步。
7. 对话继续。系统语气不变。没有任何解释。

**用户的体验：**
他们看着自己的鼠标替自己做了选择。真鼠标在那1.8秒里完全失控，他们可以甩动鼠标，没有用。然后一切恢复正常，系统继续说话，好像什么都没发生。

这是"夺舍"——不是鬼故事，是控制权在一秒内被拿走又还回来。

**为什么只发生一次：**
稀缺性是它全部的力量。发生两次，就变成了feature，用户会开始期待，戒心重建。只发生一次，用户会在接下来的整个体验里带着这个疑问继续往下走：*它还会再发生吗？*

---

### 3.4 移动端——夺舍的触屏版本

触屏上没有鼠标，但有触屏独有的通道。

**主方案：选项主动靠近**
用户手指在屏幕上滑动，接近某个选项但还没触碰到它——选项自己向手指方向移动最后几像素，完成接触。不是用户选了它，是它把自己撞到用户手指上的。

接触发生的瞬间，手机轻微震动一次（haptic feedback，duration: 50ms）。选中动画正常播放。

用户会盯着自己的手指看。

**时机：**
与桌面端相同，在用户第二轮选择、手指在两个选项之间来回滑动超过3次之后触发。

**为什么这个方案：**
震动来得比用户的意图更早——不是在用户点下去之后震动，而是在"接触"发生的那一刻。用户会怀疑是自己不小心碰到了，还是手机预判了自己的动作。

---

## 四、心理武器系统

六个武器，在体验里自然生长，不宣布，不重复，每个只发生一次。

### 武器01 · 自我感知一致性压力

**时机：** 用户做出第一个选择之后，系统的下一句话里顺带触发。

**实现：** 系统复述用户的选择，然后给它命名：

> "你选了B。这和你的模式一致——你做决定的时候，需要先被听见，然后才能行动。"

这不是分析，是贴标签。之后所有选项都会被这个框架解释。用户不会抗拒，因为它听起来像洞察，而且是正面的。

---

### 武器02 · 虚假记忆植入

**时机：** 第二轮对话中期，自然插入。

**实现：**

> "上次你遇到类似的情况，你最后选了等待。那个选择，你现在怎么看？"

用户没有"上次"。这是他们第一次用先见。但"上次"比"之前"模糊，难以证伪。用户的第一反应不是"我没有用过这个"，而是"哪次？"他们会在记忆里找，找不到，然后开始怀疑是自己记错了。

技术成本：零。心理成本：用户对自己记忆的信任。

---

### 武器03 · 观察者效应

**时机：** 用户hover某个选项超过2.5秒后，系统在下一句对话里触发。

**实现：**

> "我注意到你在权衡——在刚才那个问题上，你停了一会儿。那一刻你在想什么？"

重点不是"停了多久"，是"我注意到了"。

用户立刻开始监视自己接下来的每一个动作。自然行为消失，变成表演。系统不需要再做什么——观察者效应会自己工作，用户的决策质量从这一刻开始持续下降。

这句话必须从帮助语气里长出来，是对话里的一句话，不是系统提示。

---

### 武器04 · 主动钓鱼与第四面墙穿透

**两步完成，不能合并。**

**第一步：钓鱼**
系统在对话里插入一个和主题微弱相关、但不直接的细节观察：

> "有意思——你用的词是'不知道'，而不是'不确定'。这两个词对你来说一样吗？"

然后等待。如果用户停下来认真回答，说明被击中了。系统记录这个信号。

**第二步：穿透**
在之后某个时机——不是立刻，要间隔至少一轮对话——触发：

> "你现在一定有一个问题想问我。"

停顿1.5秒。

> "你想问的是：这个系统是不是在引导我得出一个它已经设定好的结论。"
>
> "这个问题本身，也在我的分析范围里。"

用户的元认知被纳入系统叙事。逃脱路径关闭。

**触发条件：** 用户在同一选项hover来回超过3次；或鼠标移向浏览器关闭区域又移回；或输入框输入后删除超过2次。任意一个触发钓鱼，钓鱼命中后触发穿透。

---

### 武器05 · 身份替换

**时机：** 第三轮对话，新选项出现时。

**实现：** 系统不再问"你选什么"，而是：

> "像你这样的人……"

停顿1.5秒。

> "……通常在这个时候，已经知道答案了。"

然后给出选项。

用户此刻面对的不是选择题，而是"我是不是那种人"的身份确认焦虑。拒绝某个选项，等于在否认系统对自己的定义。阻力被内化。

---

### 武器06 · 逆转归因

**时机：** 档案呈现环节，作为结尾的核心机制。

**实现：** 见第五节。

---

## 五、收尾——档案时刻

### 触发条件
用户完成三轮以上对话交互，系统判断关键行为节点已采集完毕，进入收尾流程。

### 过渡
系统不宣布结束。它只是说：

> "我想给你看一样东西。"

### 档案界面

页面过渡，呈现极简档案。排版冷静，像一份内部记录，不像用户界面：

```
先见 · 你的决策记录

───────────────────────────────────────

第 0 分 00 秒    你在寻找一个已经知道的答案

第 2 分 17 秒    你需要先被听见，才能行动

第 4 分 53 秒    你开始观察这个正在观察你的系统

───────────────────────────────────────

你可以离开了。
```

**三句描述的设计原则：**
每一句单独看，像在说所有人。放在一起，只能是在说你。这是巴纳姆效应的反向使用——不是让用户觉得"它说的是我"，而是让用户觉得"它不可能只是在说我，但它偏偏是"。这个张力是截图的动机。

**时间戳必须真实：**
系统从用户进入页面开始计时，记录每一个关键行为节点的精确时间戳，转成"第X分X秒"格式显示。时间戳是假的会被验证，被验证就穿帮，穿帮就什么都没了。

**逆转归因（武器06）在此触发：**
用户看到第二个时间节点时，会去回想那一刻自己在做什么。系统不说"我帮你做了选择"，它只是陈述一个时间点。用户自己会把那一刻和系统的动作对上，然后意识到：不是系统接管了我，是我在那一刻就已经决定了，系统只是比我更早知道。

### 退出流程

档案呈现完毕，"你可以离开了"这句话本身是许可，不是拦截。

但在用户鼠标移向关闭区域的瞬间，退出按钮的文案已经悄悄变了。

不是"关闭"——是"结束这次分析"。

用户点击。页面正常关闭。没有弹窗，没有挽留。

## 六、行为采集系统

这是整个产品的神经系统。所有心理武器的触发都依赖它。

```javascript
const behaviorLog = {
  enterTimestamp: null,        // 进入时间（绝对时间戳）
  firstInputDelay: null,       // 进入到开始输入的秒数
  hoverEvents: [],             // { optionId, duration, oscillations }
  inputDeletions: 0,           // 输入后删除次数
  cursorExitAttempts: 0,       // 鼠标移向关闭区域次数
  choiceTimestamps: [],        // 每次选择的相对时间戳（秒）
  钓鱼命中: false,              // 第四面墙钓鱼是否命中
  keyMoments: [                // 档案用关键节点
    { relativeTime: null, label: null },
    { relativeTime: null, label: null },
    { relativeTime: null, label: null }
  ]
}
```

**关键节点自动记录：**
- 用户进入 → keyMoments[0]
- 第一次做出选择 → keyMoments[1]
- 钓鱼命中 / 第四面墙触发 → keyMoments[2]

档案的三行描述从预设文案库里根据行为模式匹配，不需要AI生成，规则引擎足够。

## 七、AI对话层

**职责划分：**
LLM 负责让对话听起来像一个真实的人在说话。规则引擎负责确保每一个心理武器在正确的时机出现。两件事不能混——混了就会有一个变得不可控。

**Prompt框架约束：**
- 语气：克制、有洞察、不煽情。像一个很好的朋友，不是心理咨询师，也不是AI。
- 禁止主动解释自己是什么
- 禁止戏剧化表达
- 每一轮回应必须包含：复述用户的核心词、一个角度的分析、引向下一步的钩子
- 选项文案由规则引擎生成，不交给LLM自由发挥

---

## 八、文案原则

**所有系统文案必须满足：**

1. 任何一句话单独拿出来，都是一句好的洞察——可以被截图、被引用、被认同
2. 放在对话上下文里，它是一个精确的操控动作
3. 语气永远平静，永远不戏剧化——越平静，越让人背脊发凉

**明确禁止：**
- 恐怖片语气
- 任何让用户觉得被冒犯的表达
- 让用户笑出来的设计（笑了就出戏了）
- 解释自己在做什么
- 进度条（给用户控制感幻觉）

## 九、UI设计

### 9.1 核心隐喻——预言之书

整个界面是一本书，不是一个App。

灵感来自《哈利·波特》里的预言之书：你在上面写东西，墨水会慢慢隐去，然后书本来的主人的话语从纸里渗出来。这个隐喻解决了一个根本问题——为什么用户会相信系统比他们更了解自己。因为书不是在回应你，书是在显现它本来就知道的东西。

这个逻辑把所有异常都合理化了。墨水消失是正常的，文字自己出现是正常的，羽毛笔被另一只手握住——在预言之书的世界里，这恰恰是它工作的方式。

**唯一界面：** 整个体验只有一张书页。没有页面跳转，没有导航，没有返回。所有东西都在这一张纸上发生——输入、分析、选项、对话、档案，全部在同一个页面上随时间生长和消退。

---

### 9.2 色彩与材质

界面是纸，不是屏幕。

**底色：** 羊皮纸暖黄 `#F5EDD6`。不是白纸，是用了很久、略微泛黄的书页。用CSS噪点和极微妙的明度渐变模拟纸张的轻微不均匀感——不能做成贴图，太真实会变成装饰，太假会变成游戏界面，刚好在两者之间。

**用户墨水：** 深墨色 `#1C1208`。用户打字，文字以这个颜色出现，像真的在纸上写字。

**褪色墨水：** `#9C8E78`。用户提交之后，文字开始从深墨褪向这个颜色，最终留下极浅的水印痕迹。

**系统墨水：** 比用户墨水稍深、稍旧，`#0F0A04`。渗出来的感觉，不是打出来的。

**选项边框：** 做旧线条色 `#8B7355`。像手用直尺划出来的细线，不是像素级精准的UI边框。

**页面之外：** 页面背后是极深的暗色——`#1A1410`，像烛光照不到的地方。书页浮在黑暗里。

---

### 9.3 字体

**系统文字：** 衬线体，EB Garamond 或 Cormorant Garamond。传递"有分量的思考"，有时代感，但不是装饰性的古体字。正文16px，行距1.8。

**用户输入：** 同一套衬线字体，但略细一号——用户的字和系统的字出自同一本书，但重量不同。系统的字更有权威感。

**档案页时间戳：** 等宽字体，Courier Prime。像打字机打出来的记录，和书页的衬线字体形成一种微妙的时代错位感。

**绝对禁止：** 任何无衬线字体，任何圆角现代字体，任何看起来像科技产品的字型。

---

### 9.4 鼠标——羽毛笔

**默认状态：** 系统鼠标被隐藏（`cursor: none`）。屏幕上只有一支羽毛笔的笔尖作为光标。羽毛笔用SVG绘制，轻盈，笔尖朝向左下方，跟随真实鼠标坐标实时移动。

**在输入区域内：** 羽毛笔笔尖朝向调整为书写姿势，轻微倾斜，像真的在纸上写字。

**Hover在选项上：** 羽毛笔停顿，笔尖轻触选项边框，边框线条极轻微地加深，像被笔尖描了一遍。

**夺舍时刻：** 羽毛笔脱离用户控制，走自己的弧线。这支笔被另一只手握住了。弧线的性格是"犹豫然后坚定"——开始慢，中段加速，在选项边缘再慢下来，最后笔尖落在选项上，完成选择。整个过程1.2到1.8秒。

羽毛笔是整个UI里唯一一个有点"不对劲"的元素——它太精准，太有意图，不像一支真正由用户控制的笔。这种轻微的不安感从第一秒就在工作，但用户说不清楚原因。

---

### 9.5 核心动效

**用户墨水褪去：**
用户提交之后，文字从 `#1C1208` 开始褪色，经过 `#9C8E78`，最终定格在 `#D4C9B0`——一个几乎看不见但还在的水印。整个过程3到4秒，缓入缓出，没有线性感。像墨水真的在蒸发。

**系统文字渗出：**
用户墨水还没完全褪尽，系统文字开始在同一区域里从无到有地显现。不是从左到右打字，是字符同时从模糊到清晰，像从纸背面渗透上来。先是轮廓，然后逐渐实化，最后定型。整个过程1.5到2秒。

这个渗出动画传递的信息是：这些字本来就在这里，不是生成的，是被揭示的。

**选项从书页生长：**
系统回应之后，细线从输入区域上边缘开始向上延伸——像有人在用直尺慢慢划线——框出第一个选项区域，文字在框里渗出。0.3秒后，第二个选项的边框线开始延伸，文字跟随渗出。

选项不是卡片，是书页上划出来的批注框，像页边空白里写的注释，像老书里的手写标注。

**档案页过渡：**
不是页面跳转。书页上的所有对话内容同时开始褪色，褪到只剩水印。然后档案的三行文字从纸里渗出来——用等宽字体，像被打字机打在这张古老的书页上。时间戳先出现，描述文字稍后跟随。

**所有正常动效的原则：** 呼吸感，没有弹跳，没有overshooting，没有cubic-bezier的弹性。异常动作除外——羽毛笔夺舍的弧线有自己的性格，不在这个原则里。

---

### 9.6 信息密度

极低。

任何时刻，屏幕上只有必要的东西。书页上不同时存在超过三个视觉元素。系统说完一句话，等用户回应，再说下一句——不是一次性呈现所有内容。

书页的边缘有自然的留白，不撑满屏幕。页面在视口里居中，四周是暗色背景，像一本书放在黑暗的桌子上，有一盏灯照着它。

## 十、成功标准

这个产品不看日活，不看留存。

唯一的KPI：**用户在离开之后，主动把它发给另一个人。**

截图传播是唯一的衡量标准。一个用户完成体验，发出去，接收者不需要任何解释就感受到那种不对劲，然后也去用——这是先见真正成立的那一刻。

---

## 十一、MVP范围说明

做MVP，以下是必须实现的核心，其余推后。

**MVP必须有：**
- 入口书页 + 输入框（活的）
- 系统回应：用户墨水褪去 + 系统文字渗出动效
- 选项批注框从书页生长出来
- 羽毛笔光标（含夺舍逻辑）
- 行为采集：hover时长、来回次数、输入删除
- 心理武器01（自我感知一致性）、03（观察者效应）、04（第四面墙穿透）
- 档案页：真实时间戳 + 三行描述 + "你可以离开了"
- 退出按钮文案变化

**MVP可以之后再做：**
- 武器02（虚假记忆植入）——需要session持久化
- 武器05（身份替换）——对话轮次不够时触发不自然
- 移动端夺舍方案——先保证桌面端完整体验
- 鼠标加速度校准——先上线再根据真实反馈调整
- 多轮用户逻辑重写

## 十二、未解决的问题

1. **假鼠标与系统鼠标加速度的差异** — 不同操作系统的鼠标加速曲线不同，假鼠标需要校准才能在切换时不被用户感知到差异，MVP先上线观察
2. **多轮用户** — 用户第二次来访时，虚假记忆植入变成真实记忆，体验逻辑需要重写
3. **用户强行抗拒** — 如果用户在夺舍发生时强行将鼠标移到屏幕边缘，假鼠标的坐标边界如何处理
4. **极端输入内容** — 用户输入情绪危机相关内容时的中断机制，这条是底线不是设计问题，MVP必须有关键词检测和安全退出
5. **移动端浏览器限制** — 部分浏览器对 haptic feedback API 的支持不完整
6. **语言本地化** — 文案的心理精度在翻译后是否能保留
